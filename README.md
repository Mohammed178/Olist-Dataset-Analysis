# Olist Delivery Experience Analysis

## Hypothesis
Late deliveries cause lower review scores and reduce repeat purchase rates.

## Dataset
Brazilian e-commerce dataset (~100k orders, 2016–2018) loaded into Supabase (PostgreSQL).

## Structure
- `week1/` — SQL fundamentals: aggregations, joins, conditional logic
- `week2/` — Window functions: LAG/LEAD, ranking, cohort prep
- `week3/` — Case study queries: full delivery → review → retention analysis
- `findings.md` — Running list of insights as I go

## Key Questions
1. What is the late delivery rate, and how has it changed over time?
2. How much do late deliveries impact review scores?
3. Do customers who had a late first order buy again at lower rates?

## Tools
PostgreSQL (Supabase), Power BI (dashboard).
